Token 经济学：AI 时代的新货币战争

从硅谷的 Token-maxxing 狂潮，到中国模型的全球逆袭，再到 Token 套利的商业模式 —— 这是一场围绕"AI 时代的电力"展开的全方位战争。

1 Token-maxxing：硅谷新型"炫富"

学习目标：理解什么是 Token-maxxing 现象，以及它为何在硅谷引发激烈辩论。

在硅谷，有了一种新的"炫富"方式——每天烧了多少 token。

这个词叫 Token-maxxing，意思是"把 token 用量拉到极限"。消耗多少 token、能同时调度多少个 agent、吞吐量多高，正在变成衡量一个人"有多 AI 原生"的新指标。

2 Token-maxxing 之辩：支持 vs 反对

学习目标：了解两派观点的核心论据，并思考其中的权衡。

支持方：不用 AI 就出局

  • Writer CEO May Habib："这是生死存亡级别的问题。"她自己盯员工 token 消耗排行榜，认为不全力拥抱 AI 就会被淘汰。
  • Uber：后端系统 11% 的新代码更新已由 agent 完成，三个月前这一比例不到 1%。CTO 的愿景："把软件工程转型为 agent 软件工程。"

反对方：看结果，别看消耗

  • HubSpot CEO Yamini Rangan："Outcome maxxing >> token maxxing"——比起疯狂消耗 token，不如看实际产出。
  • Jellyfish CEO Andrew Lau："你可以一天到晚刷 token，但得到的结果可能完全不是你想要的。"
中间派共识 不充分利用 AI 的公司会被竞争对手超越并变得过时。大厂创始人和高层被卷起来，认为必须这样做才能让公司转型为更 AI 原生的公司。

3 创业公司与个人的成本焦虑

学习目标：理解 AI 时代成本结构的根本性变化。
噩耗 Uber CTO："2026 年的 AI 预算在开年几个月内就全部用完了——我以为够用一年的钱，已经没了。"主因是工程师对 Claude Code 的使用量暴增。
投资人吐槽 Chamath Palihapitiya：投资的软件公司自从用 AI 编程工具后，运营成本几个月内翻了三倍多，年化 AI 支出将达千万美元级别。"感谢 VC 们掏钱为这场 AI 盛宴买单。"

成本结构的根本性变化

+
传统 SaaS 模式
研发投入是一次性的，用户越多边际成本越低——"越卖越赚"的生意。成本可以摊薄到接近零。
+
AI 时代新模式
每个用户的每一次操作背后都有一笔 token 账单在跑。用户用得越深、功能越智能，产品方反而要付更多钱给模型厂商。成本跟着用量线性增长。

这也是为什么 token 正在变成 AI 创业公司最核心的"弹药"。有风投基金开始尝试直接给被投企业提供 token 额度，作为投资的一部分——"拿到钱第一件事就是买 token，为什么不直接给 token？"

4 拆解 Token 账单：大模型公司如何定价？

学习目标：掌握 token 定价的三层结构，理解"越贵的模型总成本可能越低"的反直觉悖论。
+
Input Token（输入）
你发给模型的所有内容——prompt、上下文、文件。这是基准价格，比值约为 1。
+
Cached Input（缓存输入）
已被系统缓存过的内容再次调用时，价格约为 Input 的 1/10（比值 0.1）。如果命中缓存，成本可以大幅降低。
+
Output Token（输出）
模型生成回答时产生的 token。价格约为 Input 的 6 倍，是账单中最大头的部分。

GPT-5 实际价格表

Token 类型GPT-5 价格GPT-5.5 短上下文

反直觉悖论 越贵的模型，可能总成本反而更低。
因为强模型一次就做对了，弱模型可能要反复重试，中间出错还需要人工介入——这些都是综合成本。token 单价虽涨，但"每个有效结论的成本"反而在下降。

Agent 场景的特殊挑战

Agent 不是一问一答，它会来回调用模型，每一步可能调用不同工具，工具又产生新的日志和上下文，返回给 agent 继续循环。最终完成任务用的 token 数，取决于任务调用了多少次 agent loop。

定价核心原则 模型定价不应按"推理成本"，而应按"模型质量与任务完成度"定价。推理能力、上下文窗口大小、完成任务的准确率和效率——这些才是定价的合理依据。

6 Token 效率革命：谁在烧冤枉钱？

学习目标：理解 Token 浪费的根源，以及 OpenRouter 和 Metronome 的关键角色。
触目惊心 当前全球企业级 AI 应用中，可能有近一半的 token 没有产生实际价值。Agent 不像人类知道"够了就停"，它会反复读取对话历史、重新扫描已处理的文件、喂入过期上下文——冗余信息滚雪球式增长。

OpenRouter：从 NFT 到 AI 的"货架之王"

  • 创始人：Alex Atallah，OpenSea 联合创始人兼 CTO。
  • 模式：统一 API 入口对接所有模型，平台抽取约 5% 费用。
  • 估值：a16z 2025 年领投 4000 万美元（估值约 5 亿美元），最新融资估值接近 13 亿美元。
  • 核心理念："供给越分散，中间商越值钱。"
  • 真正起飞：OpenClaw 爆发后，全球开发者疯狂调用各种模型驱动 Agent 工作流，需要快速切换模型的中间层。

Metronome：给 Token "装电表"

  • 客户名单：OpenAI、NVIDIA、Anthropic、Databricks 全在用。
  • 核心设计：把"发生了什么"和"该怎么收费"彻底拆开——工程团队只管上报用量数据，产品和销售团队自己配置价格。
  • 结果：累计融资约 1.28 亿美元，2026 年 1 月被 Stripe 收购。100 人团队估值追上许多模型公司。
启示 在 token 经济里，"怎么算钱"和"模型本身"一样重要——甚至更重要。计费基础设施本身就是一座金矿。

7 Token 套利：中间商如何赚差价？

• Opus = 产品经理：脑洞大、发散思维，适合做设计

8 中国 Token 出海：下一个结构性产业机会？

学习目标：理解 Token 出海作为一种全新贸易形态的逻辑和潜力。

一种全新形态的"出口"

伦敦程序员用 MiniMax 跑 OpenClaw——请求穿过海底光缆，落在贵州数据中心。GPU 开始运转，风扇开始转，电表开始跳。几秒后结果原路返回。没有一度电离开过中国电网，但这度电的价值已通过 API 账单以 token 形式完成跨境结算。

真正拉开差距的是什么？

  • 电价差距被高估：中国工业用电 0.4-0.6 元/度 vs 美国 0.8-1.2 元/度，差距没有想象中大。
  • 真正优势在基础设施响应速度：中国可以在西部沙漠铺满光伏板，通过特高压电网将电送到东部算力集群。美国不缺电，缺的是配电能力——建变电站从 330kV 降到 400V 或 800V，光是审批就"烦到爆炸"。
  • 供应链瓶颈：DDR4 DRAM 价格是一年前的 10 倍，CX7、电源、CPU 全面缺货。OpenClaw + Agent + 多模态 = "完美风暴"。
最终判断 如果上一个时代中国出口的是衬衫和家电，这个时代是电动车，下一个时代可能就是 token。Token 正在成为一种大宗商品——不需要集装箱，不需要实体商品离开国境，电力的价值通过互联网瞬间交付到全球任何开发者手上。
